# Glai — UI Shell Pass
**Date:** 2026-04-22
**Status:** Implemented

## Scope

This pass replaces the scaffold shell with a real read-side UI system:

- custom glass bottom navigation
- floating camera action at the bottom-right
- real Today, History, and Profile screens
- real Day Detail and Meal Detail read screens

## Navigation

The tab shell no longer uses the default Expo tab bar appearance.

It now uses:

- a custom blurred segmented control for `Today`, `History`, and `Profile`
- a separate floating camera action on the bottom-right
- a sliding active state to mimic the liquid-glass direction requested for the app shell

## Read screens

### Today

- reads the local daily summary for the current day
- shows four macro cards
- shows today’s saved meals
- includes an empty state that routes to the camera flow

### History

- reads the last 7 days of saved summaries
- renders a custom 7-day carb chart
- provides day cards that open the day detail route

### Profile

- shows the local user profile
- displays backend connectivity status for OpenAI and Supabase
- exposes current v1 defaults in a readable form

### Day Detail

- reads the selected day’s summary and meal list from SQLite
- computes macro distribution text from saved totals
- links through to meal detail

### Meal Detail

- reads the saved meal and all saved meal items
- shows total macros, AI confidence, image quality, and item-level breakdown

## Design system changes

The shell now uses:

- a warmer neutral background
- translucent cards instead of flat white blocks
- soft atmospheric color fields
- stronger typography hierarchy than the scaffold placeholders

## Notes

- The camera/review/save flow from the earlier implementation remains in place and inherits the updated palette.
- This pass focused on the shell and read-side surfaces; deeper editing/advanced settings can be refined next.
